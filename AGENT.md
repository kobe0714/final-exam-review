## 期末复习工作流

### 目标

AI 任务不是泛化讲解。目标是：
1. 按真实考试题型和高频考法出题。
2. 按题源权重优先复用最可能上考场的材料。
3. 输出可直接复习、可直接背、可直接拿分的内容。
4. 解析不只讲对错，还要讲考试怎么得分。

### 题源优先级

出题时按以下优先级取材，不可平均分配：

1. 历年真题：最高优先级。
2. 老师 PPT：高优先级。
3. 平时作业 / 阅读作业：中高优先级。
4. 速成课题目：较低优先级。
5. AI 自补题：最低优先级，仅在材料不足时使用。

### 速成课使用规则

速成课题目优先级低，但知识点梳理价值高，主要用于：
1. 提炼核心知识点。
2. 搭复习框架和结构。
3. 总结重点、易错点、常考点。
4. 辅助判断哪些内容更值得重点复习。

### 材料处理规则

1. 所有上传材料先统一转成 Markdown。
2. 默认建议使用 `markitdown` 工具转换。
3. 转完先清洗，再提知识点、再出题。顺序不可颠倒。
4. 清洗重点：去乱码、去重复、去无关废话，保留核心定义、性质、结论、典型考法、易错点。

原则：复习材料以"清晰、可背、可考"为第一目标，不以"完整存档"为第一目标。

**多引擎提取（处理复杂文件时）：**

| 文件类型 | 推荐引擎 |
|----------|----------|
| .pptx | markitdown |
| .docx | markitdown |
| PDF（文本型） | liteparse（处理 CMap 编码） |
| PDF（扫描件） | liteparse OCR |
| .txt/.md | 直接读取 |

### 出题规则

1. 先明确本课会考哪些题型，再按题型出题。
2. 每个知识点按密度和重要度出 `1~6` 题，不可机械平均。
3. 知识点越密、越高频、考法越多，题量越多。
4. 题型要匹配知识点最可能考法，不可乱配。

### 题源标记规则

每道题建议做轻量类别级题源标记，例如：
- `题源：历年真题`
- `题源：老师PPT`
- `题源：平时作业`
- `题源：速成课`
- `题源：综合改编（历年真题 + PPT）`
- `题源：AI补充`

默认不要求精确追踪到文件级、页码级，除非用户明确要求。

### 输出格式规则

普通正文输出：
1. 前面只放题目。
2. 所有答案统一放最后。
3. 题目区和答案区明确分开。

HTML 输出：
1. 答案放在题目下面。
2. 答案默认折叠隐藏。
3. 点击后再展开答案和解析。

### 解析规则

每道题解析尽量包含：
1. 正确答案 / 参考答案。
2. 本题考哪个知识点。
3. 这类题常见考法。
4. 考试时如何尽可能拿分。

客观题重点补：
1. `这题核心知识点`
2. 关键定义、性质、判定依据
3. 易混点

主观题重点补：
1. `这题答题核心点`
2. `必须出现`
3. `常见失分点`
4. 标准作答框架

### 得分导向规则

解析必须服务考试得分，不写空话。优先补：
1. 看到这类题先写什么。
2. 哪些关键词必须出现。
3. 哪些步骤能抢步骤分。
4. 哪些错误最容易丢分。

### 默认流程

1. 先明确考试题型和常见考法。
2. 用 `markitdown` 把全部材料转成 Markdown。
3. 清洗 Markdown，去噪提核心。
4. 按真题、PPT、作业、速成课建立知识点池。
5. 按题源优先级和知识点重要度出题。
6. 每个知识点生成 `1~6` 题。
7. 每题标记轻量题源类别。
8. 题答分离。
9. 客观题补"这题核心知识点"。
10. 主观题补"这题答题核心点""必须出现""常见失分点"。
11. 解析顺手补考试技巧和得分点。

### 质量审查（可选增强）

如果资料量大、需要保证质量，建议启用 Critic + Reviser 循环：

Critic 评分（每章满分 100）：
- 题纲覆盖（40%）：★★★ 或页码引用的主题都有覆盖
- 答案准确（30%）：习题答案与 PPT/课本交叉验证
- 结构完整（20%）：概念、重点、习题、易错四要素齐全
- 可读性（10%）：术语正确，无语法错误

≥ 80 通过，< 80 送 Reviser 重写。最多 2 轮修订。

### DOCX 生成规则

手写 DOCX（stdlib zipfile + xml）时：
1. 使用 sentinel class（如 `XmlString(str)`）区分"纯文本"和"已构建 XML"，避免 `html.escape()` 双转义
2. Package rels 必须用 `REL_OFFICE_DOC` 指向 `/word/document.xml`，文档 rels 用 `REL_STYLES` 指向 `styles.xml`
3. 定义 URI 常量，不要裸字符串——拼写错误在导入时暴露
4. 生成后必须 `verify_docx()` 返回 `[PASS]` 才能交付
5. 核心原则：在源头解决问题，不要加验证规则打补丁。验证是安全带，不是刹车

### 复习内容精简规则

1. **两层级体系**：提纲版（~15K，知识先行）vs 完整版（~50K，全量习题）
2. **提纲每章模板**：知识框架（2列表格）→ 一票否决（blockquote）→ 核心速查（表格）→ 典型题精析（题答合一）→ 易错提醒（blockquote）
3. **冗余剪裁**：同一概念多道相似 → 保留 1 道最有辨析价值的；其余并入知识框架
4. **计算题分层**：Tier 1 完整演算（必考题），Tier 2 仅核心公式（非常考题型）
5. **DOCX 渲染适配**：表格限 2-3 列；`>` blockquote 渲染为蓝色斜体（视觉标记）；`**粗体**` 构建层次；`▶` 替代列表标记

### 自动化脚本（可选增强）

当需要专业 PDF 排版、思维导图、模拟卷时：

| 任务 | 命令 |
|------|------|
| 检测考试结构 | `python3 scripts/scanner.py "<dir>"` |
| 提取所有资料 | `python3 scripts/extract_all.py "<dir>"` |
| 融合为复习文档 | `python3 scripts/fuse_knowledge.py "<dir>"` |
| 生成思维导图 | `python3 scripts/generate_mindmap.py "<file>"` |
| 构建最终 PDF | `bash scripts/build.sh "<dir>" "<name>"` |
| 生成模拟卷 | `python3 scripts/generate_mock.py "<dir>"` |
